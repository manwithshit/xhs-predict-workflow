# 🔍 XHS Predict — 发帖前的「5 双眼睛」检查

> 你评价自己内容的时候，只有一双眼睛：你自己的。
> 这个工作流给你再借 4 双。

## 背景

做内容一段时间后，我发现一个问题：

写完一篇帖子，我觉得「写得不错」的判断，其实只是**我自己视角下的「不错」**。

但看我帖子的人，是几十种不同的人——刚刷到账号的陌生人、长期关注的老读者、在评论区会说「早就知道了」的挑剔者、还有根本不在乎内容好不好只看互动信号的算法。

每一类人对同一篇内容的感受，可能完全不同。

所以我做了这个流程：**在发帖之前，让 AI 扮演 5 种不同的人，从各自的视角把帖子读一遍**，告诉我他们各自会有什么反应，以及最重要的一个问题在哪里。

---

## 5 双眼睛

| 角色 | 是谁 | 他们最在意什么 |
|------|------|--------------|
| � **刚关注的陌生人** | 刚刷到你账号、对你一无所知的人 | 第一印象：会不会停下来看？看完会不会走？ |
| 💬 **挑刺评论者** | 会在评论区说「这我早就知道了」的那种 | 内容有没有真正的增量，还是在说废话 |
| 📝 **同赛道博主** | 也在写类似内容的创作者 | 这篇和他们写的相比，差异化在哪 |
| 🎯 **铁杆用户** | 每篇都看、偶尔会互动的老读者 | 有没有带给他们新的收获，值不值得收藏分享 |
| 🤖 **算法** | 不是人，是平台的推流逻辑 | 互动信号够不够强，格式和关键词对不对 |

## 使用方式

### 方法 1：Prompt 直接用

把 `WORKFLOW.md` 里的 Prompt 复制到任意 AI 对话（Claude、ChatGPT、Gemini 等），把你的帖子内容粘贴进去即可。

### 方法 2：Antigravity / Claude Code 工作流

1. 将 `WORKFLOW.md` 复制到你的 `.agent/workflows/` 目录，重命名为 `predict.md`
2. 在对话中输入 `/predict`，粘贴草稿内容
3. 推荐插入位置：写完草稿后、正式发帖前

```
写稿 → /predict → 根据反馈修改 → 发帖
```

## 为什么这样设计

**关于「4 小时窗口」**

小红书在内容发布后有一个初始评估期，这段时间内容获得的互动质量，决定后续会不会被推给更多人。

互动的质量不是平等的。让人**主动说话**（评论）比让人**点一下**（点赞）对平台判断内容价值的影响更大。这意味着帖子结尾的互动设计——是问一个「你用 A 还是 B」的选择题，还是问一个「你遇到过什么」的开放题——会带来实质性的差距。

**关于「算法」这个角色**

算法不读内容，它读的是信号。同样一篇帖子，如果你在结尾预判了用户会想说什么、设计了让他们自然想开口的问句，算法看到的互动分布会完全不同。所以「算法」这个角色的核心任务，是检查帖子的互动设计，而不是内容质量。

## 适配自定义

这个工作流预设的目标受众是效率工具类账号（Obsidian / AI 工具方向）。

如果你的账号方向不同，修改 `WORKFLOW.md` 中的：
- **「铁杆用户」的画像描述** → 改成你真正的核心读者是谁
- **「同赛道博主」的行业背景** → 改成你所在的内容赛道

其余 3 个角色（陌生人、挑刺者、算法）对所有账号通用。
