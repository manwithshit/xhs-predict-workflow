# 🎯 XHS Predict — 小红书发布前 AI 传播预测工作流

> **在按下发布键之前，让 AI 帮你预见这篇内容的传播走势。**

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

---

## 这是什么？

一个为小红书内容创作者设计的**发布前预测工作流**，通过 5 个不同角色的 AI 评审员，模拟真实用户对你内容的反应，在发布前发现你自己看不到的盲点。

**不需要部署任何代码**。纯 Prompt 驱动，在任意 AI 对话界面（Claude、ChatGPT、Gemini 等）中触发即可。

---

## 为什么做这个？

作为小红书内容创作者，我们评价内容好不好时，通常只有一个视角：**自己**。

但实际看帖子的人有几十种不同的视角——完全不懂这个工具的路人、算法推流时的机器评估标准、竞争者的挑剔眼光、核心目标用户的真实需求……

这个工作流召集 5 个 AI 角色，各自从一个不同视角审判你的帖子，给出评分和最重要的一条修改建议。

---

## 5 席 AI 合议庭

| 评委 | 视角 | 核心问题 |
|------|------|---------|
| 👤 **普通小白** | 完全不了解这个领域的陌生用户 | 「她会点进来吗？读完会划走吗？」 |
| 🤖 **算法工程师** | 平台推流逻辑和格式规范 | 「这篇帖子算法会判定为高质量吗？」 |
| ⚔️ **竞争者** | 写过同类内容的创作者 | 「这篇比我的强吗？差异化在哪？」 |
| 🧠 **目标用户** | 你真正想留住的核心读者 | 「看完有没有学到？会不会收藏？」 |
| 😈 **魔鬼代言人** | 专门找致命弱点 | 「这篇最大的问题是什么？」 |

## 背后的平台机制

这个工作流的评审逻辑基于小红书的真实分发机制：

**CES 评分公式：**
```
CES = 点赞×1 + 收藏×1 + 评论×4 + 分享×4 + 关注×8
```
> 评论 / 分享 / 关注的权重远高于点赞。能触发对话的内容比只能触发点赞的内容价值高 4-8 倍。

**5 级流量池：**
| 流量池 | 曝光量 | 进入条件 |
|--------|--------|---------|
| L1 种子期 | 200~500 | 自动推送 |
| L2 增长期 | 1k~3k | **4小时内 CES > 50** |
| L3 目标受众 | 10k~30k | 高 CTR + 持续互动 |
| L4 破圈期 | 100k+ | 算法主动推送 |
| L5 病毒期 | 1M+ | 自发裂变 |

**关键：发布后 4 小时是决定帖子命运的最重要窗口。**

## 使用方法

### 方法 1：直接使用 Prompt

把 `WORKFLOW.md` 里的 Prompt 复制到任意 AI 对话界面，然后粘贴你的帖子内容即可。

### 方法 2：作为 Antigravity / Claude Code 工作流

如果你在使用 [Antigravity](https://antigravity.ai) 或 Claude Code：

1. 将 `WORKFLOW.md` 放入你的 `.agent/workflows/` 目录
2. 重命名为 `predict.md`
3. 在对话中输入 `/predict` 触发

### 方法 3：集成进你的发帖流程

推荐插入位置：

```
/brainstorm → /paper → /predict ← 在这里 → /publish
```

## 适配账号类型

这个工作流预设了「效率工具类」小红书账号的评委视角（核心用户是 Obsidian / AI 工具用户）。

如果你的账号方向不同，可以修改 `WORKFLOW.md` 中的：
- **评委 4「目标用户」的描述** — 改成你真正的目标读者画像
- **账号定位滤镜** — 修改你的核心关键词和赛道描述
