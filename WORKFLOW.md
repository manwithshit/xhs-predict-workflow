# /predict — 发帖前的「5 双眼睛」

## 使用方法

```
/predict                          # 读取最新草稿进行预测
/predict @[草稿文件路径]           # 对指定草稿进行预测
/predict --title "你的标题"        # 只预测标题，快速验证
```

---

## 一、核心价值

> 你写完一篇帖子，评价它好不好时，只有一个视角：你自己的。
> 这个工作流在发帖之前，让 5 种不同的人把它读一遍。

工作流位置：

```
写稿 → /predict → [根据反馈修改] → /publish
```

---

## 二、5 双眼睛

### 角色设定

| 角色 | 是谁 | 核心问题 |
|------|------|---------|
| 👋 **刚关注的陌生人** | 刚刷到你账号、对你一无所知的人 | 第一印象好不好？会不会停下来看？ |
| 💬 **挑刺评论者** | 会在评论区说「这我早就知道了」的那种 | 内容有没有真正增量？还是在说废话？ |
| 📝 **同赛道博主** | 也在写类似内容的创作者 | 差异化在哪？比同类内容强在哪？ |
| 🎯 **铁杆用户** | 每篇都看、偶尔会互动的老读者 | 有没有新收获？值不值得收藏转发？ |
| 🤖 **算法** | 不是人，是平台推流逻辑 | 互动信号够不够？格式和关键词对不对？ |

### 角色自定义说明

- **「铁杆用户」** 这个角色需要根据你的账号定位调整。默认预设是：使用 Obsidian 和 AI 工具的效率类用户，对工具有一定了解、要求较高。如果你的账号方向不同，修改这个角色的画像描述。
- 其余 4 个角色对所有账号类型通用。

## 三、工作流步骤

### Step 1: 读取内容

// turbo
1. 如果用户未指定文件，扫描 `50_Temp/drafts/` 找到最近修改的小红书草稿
2. 读取 `index.md` 中的标题和正文
3. 展示确认摘要，等待继续

```

• 字数：约 800 字
• 预计图片数：5 张
• 开始「5 双眼睛」读帖...
```

---

### Step 2: 5 双眼睛依次读帖

对每个角色，执行以下评审：

**评审 Prompt 模板：**

```
你现在扮演「{角色名称}」。

{角色描述}

请从你的视角阅读以下小红书草稿，然后：

1. 用一句话说出你读完之后的第一反应（真实的，不要客套）
2. 从你的视角评分（1-10 分），以及评分的核心理由
3. 指出你觉得这篇帖子对你来说最大的一个问题（一句话，直接说）
4. 给出你最想看到的一个改动（具体、可操作）

不需要全面评价，只说你最在意的那一点。

--- 草稿 ---
{标题}

{正文}
```

**各角色的描述：**

**👋 刚关注的陌生人：**
> 你刚在信息流里刷到这个账号，对这个人之前写过什么一无所知。你不知道「AI 工具流」「Obsidian」这些词意味着什么，你只是个在地铁上随手刷小红书的人。你代表着这篇帖子遇到的所有陌生人。

**💬 挑刺评论者：**
> 你是那种在评论区说「这有什么稀奇，我三个月前就是这样做的」的人。你已经用了很多 AI 工具和效率工具，你对内容要求很高，容易觉得内容「太浅」「太水」。你不会恶意，但你会毫不客气地指出你觉得没有价值的地方。

**📝 同赛道博主：**
> 你也在写 Obsidian 和 AI 工具相关的小红书账号，粉丝数和这个账号差不多。你专业、理性，会从「竞品对比」的视角来评估：这篇帖子有没有比你写的同类内容更好的地方？差异化在哪里？有没有让你觉得「我怎么没想到」的角度？

**🎯 铁杆用户：**
> 你关注这个账号三个月了，几乎每篇都看过。你用 Obsidian，对 AI 工具感兴趣，偶尔会评论或收藏。你是这个账号最理想的读者——你的期待值比陌生人高很多。这篇帖子有没有带给你新的东西？值不值得你花时间收藏或转发？

**🤖 算法：**
> 你不是人，你是小红书的推流逻辑。你不读内容，你读信号。你会评估：这篇帖子的结构和关键词，会让你判断它是高质量内容吗？结尾的互动设计，能不能在发布后的关键窗口期内产生足够的评论？如果互动信号不够，你不会把它推给更多人。

## 四、关于「算法」这个角色

算法不读内容，它读的是互动信号。这里有两个关键点值得注意：

**互动质量不平等**

让人主动说话（评论）比让人点一下（点赞）对平台判断内容价值的影响更大。这意味着结尾的互动设计至关重要。

- 选择题结尾：「你用 Notion 还是 Obsidian？」— 互动权重低
- 开放题结尾：「你 Obsidian 里积累最多的是什么笔记？」— 互动权重高

**发布后的关键窗口**

小红书在内容发布后有一个初始评估期，这段时间的互动质量决定了后续会不会被推给更多人。所以「算法」这个角色检查的核心，是帖子的互动设计能不能在这个窗口期内产生足够的信号。

## 五、小红书发帖注意事项

参考 `/publish` 工作流中的敏感词规则：
- 境外 AI 产品名（Claude、ChatGPT 等）需谨慎 → 用「某AI工具」替代
- 避免直接诱导互动词汇（「点赞收藏」「评论区打卡」等）
- 工具教程类内容收藏率最高，算法偏好
