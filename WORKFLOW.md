# /predict - 小红书传播预测

## 使用方法

```
/predict                          # 读取最新草稿进行预测
/predict @[草稿文件路径]           # 对指定草稿进行预测
/predict --title "你的标题"        # 只预测标题，快速验证
```

---

## 一、核心价值

> **在按下发布按钮之前，让 AI 帮你"看到"这篇内容的传播走势**

工作流位置：

```
/paper → [配图] → /predict → [根据建议优化] → /publish
```

这个工作流不是"AI 夸你写得好"——而是用**5 个扮演不同角色的 AI 评审员**，模拟真实小红书用户对你内容的反应，发现你感觉不到的盲点。

---

## 二、小红书平台机制（预测依据）

### CES 评分公式（Community Engagement Score）

```
CES = 点赞×1 + 收藏×1 + 评论×4 + 分享×4 + 关注×8
```

> 评论 / 分享 / 关注的权重远高于点赞。**能触发互动的内容比只能触发点赞的内容价值高 4-8 倍。**

### 5 级流量池机制

| 流量池 | 曝光量 | 进入条件 |
|--------|--------|----------|
| L1 种子期 | 200-500 | 自动推送 |
| L2 增长期 | 1k-3k | 4小时内 CES > 50 |
| L3 目标受众 | 10k-30k | 高点击率 + 持续互动 |
| L4 破圈期 | 100k+ | 算法主动推送 |
| L5 病毒期 | 1M+ | 自发裂变传播 |

**关键窗口：发布后 4 小时是最重要的算法评估期。**

## 三、工作流步骤

### Step 1: 读取内容

// turbo
1. 如果用户未指定文件，扫描 `50_Temp/drafts/` 找到最近修改的小红书草稿
2. 读取 `index.md` 中的标题和正文
3. 展示识别到的内容摘要，确认后进入预测

```
📋 已读取草稿：《你等了3个月的插件，我第一周就在用了》
• 字数：约 800 字
• 预计图片数：5 张
• 开始 AI 合议庭评审...
```

---

### Step 2: 5 席 AI 合议庭评审

**召集 5 个角色，各自从不同角度对内容进行评审：**

#### 🎭 评审员阵容

| 编号 | 角色 | 视角 | 权重 |
|------|------|------|------|
| 评委 A | **普通用户「小白」** | 完全不了解这个工具，只靠内容判断值不值得看 | 高 |
| 评委 B | **算法工程师** | 从平台规则看：关键词、互动诱导、格式合规 | 中 |
| 评委 C | **同类题材的竞争者** | 这个题材我也发过，这篇能打败我吗？ | 中 |
| 评委 D | **目标受众「效率控」** | 我是用 Obsidian 的人，这篇对我有用吗？ | 高 |
| 评委 E | **魔鬼代言人** | 专门找这篇内容最致命的弱点 | 中 |

#### 📋 评审提示词

对每个评审员，按以下结构执行评估：

```
你现在扮演 [角色描述]。

请从你的角度阅读以下小红书草稿，然后：
1. 给出你的第一反应（2-3 句）
2. 评估以下 5 个维度（每个 1-10 分）：
   - 封面标题吸引力：4小时内会有多少人点进来？
   - 内容价值感：读完之后觉得"学到了"的概率？
   - 互动触发力：会让人评论/收藏/分享的冲动有多强？
   - 平台适配度：是否符合小红书算法偏好的内容形态？
   - 人设一致性：与「职场懒人」账号定位是否匹配？
3. 指出最大的一个问题（一句话，直接说）
4. 给出最重要的一条优化建议（一句话，具体可操作）

--- 草稿内容 ---
{标题} + {正文}
```

## 四、评审逻辑补充说明

### 什么时候会给出「建议重写」？

- 标题没有触发好奇心的诱因（5 分以下）
- 内容读完没有"可学的东西"（价值感 < 5）
- 与账号定位严重偏离（人设一致性 < 4）
- 竞争者评委认为"这篇内容比我弱"

### 小红书特有注意事项

参考敏感词规则（同 `/publish` 的 Step 2.5）：
- 不直接点名 Claude / ChatGPT → 用「某AI工具」代替
- 避免诱导互动词汇（点赞、收藏、评论区打卡等直白表达）
- 工具教程类最受算法偏好（对比观点类）

### 账号定位滤镜

「职场懒人」账号的核心人群：
- **效率工具用户**：用 Obsidian、Claude、自动化工具提升工作效率
- **内容偏好**：干货教程 > 工具测评 > 观点类
- **关键词锚定**：Obsidian、AI、工作流、自动化、效率

## 六、注意事项

1. **预测不等于保证**：AI 是模拟评委，不是算法本身，用于发现盲点而非精确预测数据
2. **每次发布前都应该跑一遍**：即使你觉得写得很好，评委视角往往能发现你看不到的问题
3. **P0 建议是必改的**：发现致命问题但坚持发布，风险自担
4. **标题是最值得优化的**：CTR 差距可以是 2-3 倍，优先保证标题通过测试
